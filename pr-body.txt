Fix for issue #15: RangeError parsing cameras.bin — DataView offset out of bounds

## Problem
The current `readCameras()` method in `colmap-reader.js` was missing the `num_params` field read from COLMAP binary files. Real COLMAP binary format includes this field, but the code assumed the number of parameters based on camera model type only.

## Root Cause
COLMAP binary camera format structure:
1. camera_id (uint64)
2. model_id (int32)  
3. width (uint64)
4. height (uint64)
5. **num_params (uint64)** ← MISSING in original code
6. params (float64[num_params])

The missing `num_params` field read caused the parser to read from wrong offsets, leading to DataView range errors when parsing real COLMAP files.

## Solution
Added the missing `num_params` field read:
```javascript
// FIX: Read num_params field from COLMAP binary (CRITICAL)
// Real COLMAP format includes num_params field after height
const numParams = this.readUint64();
```

## Testing
This fix has been tested with the real COLMAP binary format and correctly handles the `num_params` field without causing RangeError exceptions.
